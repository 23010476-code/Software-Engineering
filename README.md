# Software-Engineering
# Giới thiệu :
Project web bán hàng là một hệ thống website được xây dựng nhằm hỗ trợ việc trưng bày, quản lý và bán sản phẩm trực tuyến. Hệ thống cho phép người dùng xem sản phẩm, tìm kiếm, thêm vào giỏ hàng, đặt mua và theo dõi đơn hàng một cách thuận tiện. Đồng thời, admin quản trị có thể quản lý sản phẩm, danh mục, khách hàng và đơn hàng, giúp quá trình kinh doanh diễn ra hiệu quả hơn. Project này hướng đến mục tiêu nâng cao trải nghiệm mua sắm online và tối ưu hóa hoạt động quản lý bán hàng.
# Phân công nhiệm vụ :
Phân công 4 người

🔹 Người 1 – Frontend (UI/UX)
Nhiệm vụ:
Thiết kế giao diện (HTML, CSS)
Trang chính: trang chủ, danh sách sản phẩm
Trang chi tiết sản phẩm
Giỏ hàng (UI)
👉 Kết quả:
Giao diện hoàn chỉnh, responsive
Có thể dùng Bootstrap/Tailwind cho nhanh

🔹 Người 2 – Backend (User + Auth)
Nhiệm vụ:
Xây dựng API đăng ký / đăng nhập
Xử lý xác thực (login, session/JWT)
API user (lấy thông tin người dùng)
👉 Kết quả:
Login/logout chạy được
Kết nối frontend

🔹 Người 3 – Backend (Product + Order)
Nhiệm vụ:
API sản phẩm (CRUD)
API giỏ hàng
API đặt hàng
👉 Kết quả:
Thêm/xóa sản phẩm
Đặt hàng thành công

🔹 Người 4 – Database + Documentation (rất quan trọng)
Nhiệm vụ:
Thiết kế database (ERD)
Tạo bảng (MySQL/MongoDB)
Viết tài liệu:
URS
SRS
Use Case Diagram
Sequence Diagram

# Mô hình hoá dự án :
1. Khảo sát yêu cầu
→ Tìm hiểu nhu cầu của người dùng và quản trị viên
→ Xác định chức năng cần có: đăng nhập, xem sản phẩm, giỏ hàng, đặt hàng, thanh toán, quản lý đơn hàng
2. Phân tích hệ thống
→ Phân tích tác nhân tham gia: khách hàng, quản trị viên
→ Phân tích dữ liệu và luồng xử lý của hệ thống
3. Thiết kế hệ thống
→ Thiết kế giao diện web
→ Thiết kế cơ sở dữ liệu
→ Thiết kế các chức năng chính và sơ đồ hoạt động
4. Xây dựng chương trình
→ Lập trình giao diện người dùng
→ Lập trình chức năng đăng ký, đăng nhập, giỏ hàng, đặt hàng, quản lý sản phẩm
→ Kết nối với cơ sở dữ liệu
5. Kiểm thử hệ thống
→ Kiểm tra lỗi giao diện
→ Kiểm tra chức năng hoạt động đúng hay sai
→ Sửa lỗi phát sinh
6. Triển khai và vận hành
→ Đưa website lên môi trường thật
→ Cho người dùng sử dụng thử và chính thức sử dụng
7. Bảo trì và nâng cấp
→ Sửa lỗi sau triển khai
→ Cập nhật thêm chức năng mới
→ Tối ưu hiệu năng hệ thống
